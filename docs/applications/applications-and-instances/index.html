アプリケーションとインスタンス 編集

すべてのEmberアプリケーションは、Applicationクラスを継承しています。 このクラスは、アプリケーションを構成する多くのオブジェクトの宣言と設定に使います。

アプリケーションの起動時に、ステートフルなものの管理のためにApplicationInstanceが作成されます。 このインスタンスは、アプリケーション用にインスタンス化されたオブジェクトのowner(オーナー、所有者)として機能します。

基本的に、Applicationはアプリケーションを定義し、ApplicationInstanceはアプリケーションの状態を管理します。

こうした関心の分離は、アプリのアーキテクチャを明確にするだけでなく、効率を向上させることもできます。 これは、テストやサーバーレンダリング(例えば、FastBoot経由)中にアプリケーションを繰り返し起動する必要がある場合に特に当てはまります。 Applicationの設定は一度だけ可能で、複数のステートフルなApplicationInstanceインスタンス間で共有することができます。 これらのインスタンスは、不要になったら破棄することができます(例えば、テスト実行後やFastBootリクエストが終了した時など)。
